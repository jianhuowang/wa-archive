## 一、从一个具体的题开始

> 你在坐标 (x, y)，表示两个方向分别还剩 x 格和 y 格可以走。每步**选一个方向**（横或纵），把对应坐标减少 **1 或 2**（不能减成负数）。谁先没法走——即被堵在 (0,0)——谁输。问先手赢还是输？

### 1.1 先简化成一维：只剩一个方向

站在位置 n（还剩 n 步能走）。每步走 1 或 2。走到 0 动不了 → 输。

怎么判断输赢？一个位置是**输**，当且仅当：**你能走到的所有位置，全是赢**（你怎么走，对手都笑）。反之是赢，当且仅当：**能走到的位置里至少有一个是输**（你走那步把必败甩给对手）。

```
n=0: 动不了                           → 输
n=1: 只能走到 0（输）                  → 赢
n=2: 能走到 1（赢）、0（输）           → 赢（走 2 步到 0）
n=3: 能走到 2（赢）、1（赢）           → 输（怎么走对面都赢）
n=4: 能走到 3（输）、2（赢）           → 赢（走 1 步到 3）
n=5: 能走到 4（赢）、3（输）           → 赢（走 2 步到 3）
n=6: 能走到 5（赢）、4（赢）           → 输
```

规律：**n % 3 == 0 时是输。**

### 1.2 引入 SG 值

用一个数标记每个位置——**SG 值**。

```
SG(n) = mex{ 从 n 能走到的所有位置的 SG 值 }
```

**mex** = 从 0 往上数，第一个**不在**这个集合里的非负整数。

```
mex{}      = 0    （空集合，0 不在 → 答 0）
mex{0}     = 1    （0 在 → 往上，1 不在 → 答 1）
mex{0,1}   = 2    （0 在, 1 在, 2 不在 → 答 2）
mex{1,2}   = 0    （0 不在！→ 答 0）
mex{0,2}   = 1    （0 在, 1 不在 → 答 1）
```

逐行推算一维网格的 SG：

| n | 能走到 | 那些位置的 SG | mex | SG |
|---|--------|--------------|-----|----|
| 0 | — | — | mex{} | 0 |
| 1 | 0 | {0} | mex{0} | 1 |
| 2 | 1, 0 | {1, 0} | mex{1,0} | 2 |
| 3 | 2, 1 | {2, 1} | mex{2,1} | 0 |
| 4 | 3, 2 | {0, 2} | mex{0,2} | 1 |
| 5 | 4, 3 | {1, 0} | mex{1,0} | 2 |
| 6 | 5, 4 | {2, 1} | mex{2,1} | 0 |

SG = 0 → 必败。SG ≠ 0 → 必胜。序列：0, 1, 2, 0, 1, 2, … → 循环 = 3。

**SG(n) = n % 3。**

### 1.3 回到二维

两个方向互不影响 → 两个独立子游戏。一维 SG(x) = x % 3，一维 SG(y) = y % 3。

**SG 定理**：独立子游戏的总 SG = 各子游戏 SG 的异或（XOR）。

```
SG(总) = (x % 3) ⊕ (y % 3)
```

mod 3 只有 0、1、2。XOR = 0 的唯一情况是两个数相等。

```
(x % 3) == (y % 3)  →  必败
等价于 (x - y) % 3 == 0
```

**答案：`(x - y) % 3 == 0` 先手必败，否则必胜。**

## 二、从例题到一般规律

### 2.1 单堆 SG —— 步数是连续区间 [a, b]

循环节 = **a + b**。

**公式**：`SG(n) = (n % (a + b)) / a`（整数除法）。

以 {2, 3, 4} 为例（a=2, b=4）：

```
n=0: 不能走 (0<2) → mex{} = 0
n=1: 不能走 (1<2) → mex{} = 0
n=2: 只能走 2→0    → mex{SG(0)} = mex{0} = 1
n=3: 走 2→1, 3→0   → mex{SG(1), SG(0)} = mex{0,0} = 1
n=4: 走 2→2, 3→1, 4→0 → mex{SG(2), SG(1), SG(0)} = mex{1,0,0} = 2
n=5: 走 2→3, 3→2, 4→1 → mex{SG(3), SG(2), SG(1)} = mex{1,1,0} = 2
n=6: 走 2→4, 3→3, 4→2 → mex{SG(4), SG(3), SG(2)} = mex{2,1,1} = 0 ← 回到 0
```

SG 序列：`0, 0, 1, 1, 2, 2, 0, 0, ...`。n = 0..5 与 n = 6..11 完全一致 → 循环 = 6 = a + b。

代入公式验证：`SG(4) = (4 % 6) / 2 = 4 / 2 = 2` ✓，`SG(7) = (7 % 6) / 2 = 1 / 2 = 0` ✓。

速查：

| 步数 | a+b | SG 规律 |
|------|-----|---------|
| {1, 2} | 3 | n%3 — 即 `(n%3)/1` |
| {1, 2, 3} | 4 | n%4 |
| {2, 3, 4} | 6 | 0,0,1,1,2,2 |
| {3, 4, 5} | 8 | 0,0,0,1,1,1,2,2,2 → `(n%8)/3` |

> **注意**：循环 = a+b，不是 max+1。{2,3,4} 的 max+1=5 ≠ 6。

### 2.2 单堆 SG —— 步数不连续（如 {1, 3, 4}）

没有公式，从 n=0 硬推。

```
n=0: {}                     → mex{}     = 0
n=1: {SG(0)} = {0}          → mex{0}   = 1
n=2: {SG(1)} = {1}          → mex{1}   = 0
n=3: {SG(2), SG(0)}         → mex{0,0} = 1
n=4: {SG(3), SG(1), SG(0)}  → mex{1,1,0} = 2
n=5: {SG(4), SG(2), SG(1)}  → mex{2,0,1} = 3
n=6: {SG(5), SG(3), SG(2)}  → mex{3,1,0} = 2
n=7: {SG(6), SG(4), SG(3)}  → mex{2,2,1} = 0  ← SG 再次为 0
```

序列：`0,1,0,1,2,3,2,0,...`

**怎么判断循环？** SG(n) 仅依赖前 maxStep=4 个值——因为 mex 往回看不会超过 maxStep 步。窗口 `<SG(n-4), SG(n-3), SG(n-2), SG(n-1)>` 的状态空间有限（每个位置 SG ∈ [0, maxStep]），必然重复。**当某个窗口跟之前某个窗口完全一样**时，从该点起整段序列确定循环。不能看到单个 SG 值重复就说循环——要确认窗口/整段模式的一致重复。

这里 n=7 的 SG=0，但 n=7 的窗口 `<SG(3),SG(4),SG(5),SG(6)> = <1,2,3,2>` 与 n=0 的窗口不重合（n=0 前没有 4 个值）。需要继续推直到窗口也重复。比赛里步数集不大时直接跑前 200 项然后 detectCycle。

## 三、多堆独立子游戏 → XOR

### 3.1 为什么是 XOR

**SG 定理**：独立子游戏的总 SG = 各子游戏 SG 的异或。

### 3.2 XOR = 0 的本质

XOR = 二进制逐列数 1 的个数，不进位：

**非零例** `0 ⊕ 1 ⊕ 2`：
```
0 = 00
1 = 01
2 = 10
⊕ = 11 = 3
第 0 位：0 + 1 + 0 = 1（奇数 → 1）
第 1 位：0 + 0 + 1 = 1（奇数 → 1）
→ XOR ≠ 0 → 有得修 → 必胜
```

**归零例** `1 ⊕ 2 ⊕ 3`：
```
1 = 01
2 = 10
3 = 11
⊕ = 00 = 0
第 0 位：1 + 0 + 1 = 2（偶数 → 0）
第 1 位：0 + 1 + 1 = 2（偶数 → 0）
→ XOR = 0 → 平衡
```

- **每列 1 的个数全为偶数** → XOR = 0 → **平衡态** → 必败
- **存在某列 1 的个数为奇数** → XOR ≠ 0 → 可以修 → 必胜

### 3.3 如何修复不平衡

当前总 XOR = k ≠ 0。找 k 的**最高位 1**所在的列 → 必有一堆在那列也是 1，设该堆 SG = g。

目标：让该堆 SG 从 g 变成 `g' = g ⊕ k`。因为 k 的最高位 1 来自 g，所以 g' < g。

**关键**：SG 是 mex 定义出来的标签，不是物理状态本身。但 mex 的定义保证了——若某状态的 SG = g，则它**一定存在后继状态**，其 SG 为 0, 1, 2, …, g-1 中的任意值。既然 g' < g，就存在一个合法走法，让该子游戏的 SG 精确变成 g'。

修改后新 XOR = k ⊕ g ⊕ g' = k ⊕ g ⊕ (g ⊕ k) = 0。平衡态甩给对手。

### 3.4 正确步骤

**先取余（算 SG），再 XOR。** 别跳过这步直接 XOR 原数。

## 四、Misère Nim（最后一个拿的**输**）

规则反转：拿最后一个人**输**。

Misère Nim **不能**简单写"有大堆时跟标准 Nim 一样"。胜负条件可以用 XOR 近似判断，但**策略必须区分情况**。

### 情况一：所有堆都是 1

退化为奇偶博弈：
- **奇数堆** → 先手输（你被迫拿最后一个）
- **偶数堆** → 先手赢（对手拿最后一个）

### 情况二：至少两堆 ≥ 2

策略跟标准 Nim 基本一致——按 XOR 走。双方会自然避开采最后一颗的坑，因为有多堆缓冲。

### 情况三：只有一堆 ≥ 2，其余全是 1

设 k 堆大小为 1，外加一个大堆。

- **k 为奇数**：**拿光大堆**。剩下奇数个 1 → 对手被迫拿最后一个。
- **k 为偶数**：**把大堆变成 1**。剩下奇数个 1 → 对手被迫拿最后一个。

**反例警示**：[1,2]，k=1（奇数），大堆=2。

标准 Nim 会算 XOR = 1⊕2 = 3 ≠ 0，找 2 的最高位修：2⊕3 = 1，把 2 变成 1 → [1,1]。但 [1,1] 是先手面对偶数个 1 → **先手拿一个，对手拿最后一个，先手赢了？不是——先手拿了倒数第二个被对手捡了最后一个。** 实际上 [1,1] 谁先动谁输（先手拿一个，对手拿最后一个——先手赢？不，Misère 下拿最后一个输，对手拿最后一个 → 对手输，先手赢。但等等……）

慢着，再推一遍 [1,2]：
- 标准 Nim 修成 [1,1]。
- [1,1] 先手（对手）面对：拿一个 → 剩 [0,1] → 后手（你）被迫拿最后一个 → **你输**。

所以标准 Nim 的修法在 Misère 下是送死。

正确走法：k=1（奇数），拿光大堆 2 → 剩 [1]。对手拿最后一个 → **对手输，你赢。**

**记忆口诀**：只有一堆大的、其余全 1 时，**目标是把局面变成奇数个 1 甩给对手**。k 奇 → 拿光大堆（k 不变，仍奇数）；k 偶 → 大堆变 1（k+1 变奇数）。

## 五、练习：检验一下

> 三堆 a=3, b=4, c=5。每堆取 1 或 2。取走最后一个赢。先手？

```
SG(3) = 0, SG(4) = 1, SG(5) = 2
0 ⊕ 1 ⊕ 2 = 3 ≠ 0 → 先手赢
解法：c 取 1 → c=4 → SG=1 → 0⊕1⊕1 = 0 ✓
```

> 两堆 [1,2]。取走最后一个**输**。先手怎么走？

```
k=1（奇数），大堆=2。拿光大堆 → 剩 [1]。对手被迫拿最后一个 → 先手赢。
```

## 六、速查卡片

| 概念 | 一句话 |
|------|--------|
| SG 值 | mex{能走到的所有位置的 SG} |
| SG = 0 | 必败 |
| SG ≠ 0 | 必胜 |
| mex | 从 0 往上数，第一个不在集合里的数 |
| 连续步数 [a,b] | 循环 = a+b；SG(n) = (n % (a+b)) / a |
| 离散步数 | 硬推，窗口状态有限 → 最终循环；判断循环看窗口/整段重复 |
| 多堆独立 | 先算各 SG，再 XOR |
| XOR = 0 | 每列 1 个数全为偶数 = 平衡 |
| XOR ≠ 0 修复 | SG=g 能走到 0..g-1，取 g'=g⊕k < g 即可 |
| Misère | 全 1 → 奇偶判断；至少两堆 ≥ 2 → 接近标准 Nim；一堆 ≥ 2 + 其余全 1 → 留给对手奇数个 1 |
