# 单堆 SG 的计算规律

## 连续步数 [a, b]

每步可取 a, a+1, …, b 个（连续区间）。

**循环节 = a + b**。公式：

$$
SG(n) = \left\lfloor \frac{n \bmod (a+b)}{a} \right\rfloor
$$

### 示例：{2, 3, 4}（a=2, b=4）

```
n=0: 不能走 → mex{} = 0
n=1: 不能走 → mex{} = 0
n=2: 走 2→0 → mex{SG(0)} = mex{0} = 1
n=3: 走 2→1, 3→0 → mex{SG(1), SG(0)} = mex{0,0} = 1
n=4: 走 2→2, 3→1, 4→0 → mex{SG(2), SG(1), SG(0)} = mex{1,0,0} = 2
n=5: 走 2→3, 3→2, 4→1 → mex{SG(3), SG(2), SG(1)} = mex{1,1,0} = 2
n=6: 走 2→4, 3→3, 4→2 → mex{SG(4), SG(3), SG(2)} = mex{2,1,1} = 0
```

SG 序列：`0, 0, 1, 1, 2, 2, 0, 0, ...` → 循环 = 6 = 2+4。

验证公式：`SG(4) = (4 % 6) / 2 = 2 ✓`，`SG(7) = (7 % 6) / 2 = 0 ✓`。

### 速查

| 步数 | a+b | SG 规律 |
|------|-----|---------|
| {1, 2} | 3 | n % 3 |
| {1, 2, 3} | 4 | n % 4 |
| {2, 3, 4} | 6 | 0,0,1,1,2,2 重复 |
| {3, 4, 5} | 8 | 0,0,0,1,1,1,2,2,2 → `(n%8)/3` |

> 循环 = **a+b**，不是 max+1。{2,3,4} 的 max+1=5 ≠ 6。{1,2} 碰巧 a+b = max+1 才容易误判。

## 离散步数（如 {1, 3, 4}）

没有通用公式，从 n=0 开始硬推：

```
n=0: {}                     → mex{}     = 0
n=1: {SG(0)}                → mex{0}   = 1
n=2: {SG(1)}                → mex{1}   = 0
n=3: {SG(2), SG(0)}        → mex{0,0} = 1
n=4: {SG(3), SG(1), SG(0)} → mex{1,1,0} = 2
n=5: {SG(4), SG(2), SG(1)} → mex{2,0,1} = 3
n=6: {SG(5), SG(3), SG(2)} → mex{3,1,0} = 2
n=7: {SG(6), SG(4), SG(3)} → mex{2,2,1} = 0
```

SG 序列：`0,1,0,1,2,3,2,0,...`

### 怎么判断循环？

SG(n) 仅依赖前 maxStep 个值（最大步长 = 4 时就只看前 4 个）。窗口 `<SG(n-4), SG(n-3), SG(n-2), SG(n-1)>` 的状态空间有限（每个位置 SG ≤ maxStep），**必然重复**。

**关键**：不能看到单个 0 就以为循环。要确认**整个窗口**重复才确定循环。比赛里集合小，直接跑前 200 项然后 detectCycle。
