# 右端点固定的区间 gcd 种类数

> 📎 独立知识点：[[GCD的合并性质]]

---

## 0. 本题核心

这题最关键的不是 ST 表，而是这个状态转移：

> 已经知道所有以 `i-1` 为右端点的后缀 gcd，
> 那么所有以 `i` 为右端点的后缀 gcd，可以由前者统一接上 `a[i]` 得到。

也就是：

$$
\gcd(a_j,a_{j+1},\cdots,a_i) = \gcd(\gcd(a_j,a_{j+1},\cdots,a_{i-1}),a_i)
$$

所以这题有很强的 DP 味道：

```cpp
g[i] = 由 g[i-1] 转移得到
```

其中 `g[i]` 存的是所有以 `i` 为右端点的后缀 gcd 的压缩结果。

---

## 1. 原题

给定一个长度为 $n$ 的数组：

$$
a=[a_1,a_2,\cdots,a_n]
$$

对数组进行多次查询。每次查询一个连续区间 $[l,r]$，需要计算集合：

$$
S=\bigcup_{i=l}^{r}\left\{\gcd_{j=i}^{r} a_j\right\}
$$

的大小。

其中：

$$
\gcd_{j=p}^{q}a_j
$$

表示：

$$
a_p,a_{p+1},\cdots,a_q
$$

的最大公因数。

换成人话，对于一次查询 $[l,r]$，要求的是：

$$
\gcd(a_l,a_{l+1},\cdots,a_r)
$$

$$
\gcd(a_{l+1},a_{l+2},\cdots,a_r)
$$

$$
\gcd(a_{l+2},\cdots,a_r)
$$

一直到：

$$
\gcd(a_r)
$$

这些值中有多少种不同的 gcd。

注意：这题不是问 $[l,r]$ 内所有子区间的 gcd，而是**右端点固定为 $r$，左端点从 $l$ 到 $r$ 变化**。

### 数据范围

$$
1 \le n,m \le 6\times 10^5
$$

$$
1 \le a_i \le 2^{31}
$$

注意：

$$
2^{31}=2147483648
$$

已经超过普通 `int` 上限，所以数组值和 gcd 值必须用 `long long`。

### 样例

输入：

```cpp
6 3
1 2 3 4 5 6
1 3
2 4
1 6
```

输出：

```cpp
2
2
2
```

例如查询 $[1,3]$：

$$
\gcd(1,2,3)=1
$$

$$
\gcd(2,3)=1
$$

$$
\gcd(3)=3
$$

所以集合为：

$$
{1,3}
$$

答案是 $2$。

---

## 2. 重点一：为什么 `g[i]` 可以由 `g[i-1]` 转移？

这是本题的重中之重。

定义：

$$
f_i(j)=\gcd(a_j,a_{j+1},\cdots,a_i)
$$

表示右端点为 $i$，左端点为 $j$ 的后缀 gcd。

那么对于 $j=i$：

$$
f_i(i)=a_i
$$

对于 $j<i$：

$$
f_i(j)=\gcd(f_{i-1}(j),a_i)
$$

因为：

$$
\gcd(a_j,\cdots,a_{i-1},a_i) = \gcd(\gcd(a_j,\cdots,a_{i-1}),a_i)
$$

所以所有以 $i$ 为右端点的后缀 gcd，可以由所有以 $i-1$ 为右端点的后缀 gcd 接上 $a_i$ 得到。

也就是说：

```cpp
以 i 结尾的后缀 gcd
=
单独的 a[i]
+
以 i-1 结尾的后缀 gcd 全部和 a[i] 再 gcd 一遍
```

这就是本题的 DP 转移。

---

## 3. 重点二：为什么可以只存压缩后的 `g[i]`？

如果暴力存所有后缀 gcd，那么 `g[i]` 可能长这样：

```cpp
gcd(a[i])
gcd(a[i-1], a[i])
gcd(a[i-2], a[i-1], a[i])
...
gcd(a[1], ..., a[i])
```

这显然总量会爆。

但 gcd 有一个重要性质：

> 固定右端点，左端点往左扩展时，gcd 只会不变或变成当前 gcd 的因子。

因此相同 gcd 会连续出现。

例如：

```cpp
a = [12, 18, 6, 30]
```

固定右端点 $4$：

```cpp
起点 4: gcd(30)          = 30
起点 3: gcd(6,30)        = 6
起点 2: gcd(18,6,30)     = 6
起点 1: gcd(12,18,6,30)  = 6
```

所以完整序列是：

```cpp
起点:  4   3   2   1
gcd : 30   6   6   6
```

可以压缩成：

```cpp
{30, 4}
{6, 3}
```

含义是：

```cpp
从起点 4 开始，出现 gcd = 30
从起点 3 开始，gcd 变成 6
```

所以 `g[i]` 不存所有左端点，只存 gcd 发生变化的位置。

---

## 4. 状态定义

令：

```cpp
g[i]
```

表示所有以 `i` 为右端点的后缀 gcd 的变化点。

每个元素是：

```cpp
{val, id}
```

含义：

```cpp
从起点 id 开始，后缀 gcd 出现了 val 这个新值
```

例如：

```cpp
g[4] = {
    {30, 4},
    {6, 3}
};
```

表示：

```cpp
起点 4 的 gcd 是 30
起点 3、2、1 的 gcd 是 6
```

注意：`g[i]` 中的 `id` 是从大到小的，也就是从右往左存。

---

## 5. 预处理转移

处理第 `i` 个位置时，先加入只包含自己的后缀：

```cpp
g[i].push_back({a[i], i});
```

然后遍历 `g[i-1]`。

如果 `g[i-1]` 中有：

```cpp
{x, id}
```

表示：

$$
\gcd(a_{id},a_{id+1},\cdots,a_{i-1})=x
$$

现在右端点从 `i-1` 扩展到 `i`，右边多接一个 `a[i]`，新的 gcd 是：

$$
\gcd(x,a_i)
$$

所以代码是：

```cpp
for (auto [x, id] : g[i - 1]) {
    ll v = gcd(x, a[i]);

    if (g[i].back().first != v) {
        g[i].push_back({v, id});
    }
}
```

为什么只需要判断 `g[i].back().first != v`？

因为相同 gcd 一定连续出现。
如果新算出来的 `v` 和上一个 gcd 一样，就说明没有发生新的变化，不需要再存一次。

---

## 6. 转移例子

数组：

```cpp
a = [12, 18, 6]
```

### 处理 `i = 1`

只有：

```cpp
gcd(12) = 12
```

所以：

```cpp
g[1] = {
    {12, 1}
};
```

---

### 处理 `i = 2`

先加入自己：

```cpp
{18, 2}
```

然后由 `g[1]` 转移：

```cpp
{12, 1} -> gcd(12, 18) = 6
```

所以：

```cpp
g[2] = {
    {18, 2},
    {6, 1}
};
```

含义：

```cpp
起点 2: gcd(18) = 18
起点 1: gcd(12,18) = 6
```

---

### 处理 `i = 3`

先加入自己：

```cpp
{6, 3}
```

然后由 `g[2]` 转移：

```cpp
{18, 2} -> gcd(18, 6) = 6
{6, 1}  -> gcd(6, 6)  = 6
```

理论上会得到：

```cpp
{6, 3}
{6, 2}
{6, 1}
```

但它们的 gcd 值都一样，只保留最靠右的变化点：

```cpp
g[3] = {
    {6, 3}
};
```

含义：

```cpp
起点 3、2、1 的 gcd 全都是 6
```

这说明压缩状态不会丢失信息。

## 7. 查询怎么回答？

一次查询 $[l,r]$ 要求：

```cpp
gcd(l, r)
gcd(l+1, r)
...
gcd(r, r)
```

右端点固定为 `r`，所以所有 gcd 变化点都已经存在 `g[r]` 里。

答案就是：

```cpp
g[r] 中 id >= l 的元素数量
```

例如：

```cpp
g[4] = {
    {30, 4},
    {6, 3},
    {1, 1}
};
```

查询 `[2,4]`：

```cpp
{30, 4} 算，因为 4 >= 2
{6, 3}  算，因为 3 >= 2
{1, 1}  不算，因为 1 < 2
```

答案是 `2`。

因为 `id` 是递减的，所以查询时可以提前退出：

```cpp
int ans = 0;

for (auto [val, id] : g[r]) {
    if (id < l) break;
    ans++;
}
```

---

## 8. 为什么不会超时？

对于固定右端点，gcd 每次真正变化时，会变成当前 gcd 的真因子。

如果：

$$
y \mid x
$$

且：

$$
y \ne x
$$

那么：

$$
y \le \frac{x}{2}
$$

所以 gcd 每发生一次变化，至少减半。

本题：

$$
a_i \le 2^{31}
$$

因此对于每个右端点，gcd 最多变化约 $31$ 次。

也就是说：

```cpp
g[i].size() = O(log A)
```

其中 $A$ 是值域。

所以预处理复杂度：

$$
O(n\log A)
$$

每次查询复杂度：

$$
O(\log A)
$$

总复杂度：

$$
O((n+m)\log A)
$$

本题中：

$$
n,m \le 6\times 10^5
$$

$$
\log A \approx 31
$$

可以通过。

## 9. C++ 代码

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;
using pii = pair<ll, int>;

void solve() {
    int n, m;
    cin >> n >> m;

    vector<ll> a(n + 1);
    vector<vector<pii>> g(n + 1);

    for (int i = 1; i <= n; i++) {
        cin >> a[i];
    }

    for (int i = 1; i <= n; i++) {
        g[i].push_back({a[i], i});

        for (auto [x, id] : g[i - 1]) {
            ll v = gcd(x, a[i]);

            if (g[i].back().first != v) {
                g[i].push_back({v, id});
            }
        }
    }

    while (m--) {
        int l, r;
        cin >> l >> r;

        int ans = 0;

        for (auto [val, id] : g[r]) {
            if (id < l) break;
            ans++;
        }

        cout << ans << '\n';
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    solve();

    return 0;
}
```

## 10. 类型坑

题目给的是：

$$
1 \le a_i \le 2^{31}
$$

而普通 `int` 最大值是：

$$
2^{31}-1
$$

所以 `a_i` 可能爆 `int`。

正确类型分配：

```cpp
int n, m;                 // 下标范围，int 够
int l, r;                 // 查询下标，int 够
vector<ll> a;             // 数组值，必须 long long
pair<ll, int>             // first 是 gcd 值，second 是位置
```

推荐：

```cpp
using ll = long long;
using pii = pair<ll, int>;
```

不要无脑使用：

```cpp
#define int long long
```

因为它会把所有下标、数组大小、pair、ST 表等也变成 `long long`，可能导致内存翻倍或者位运算出问题。

## 11. 这道题学到了什么

### 1. 读题要抓住"右端点固定"

这题不是求 $[l,r]$ 内所有子区间 gcd，而是：

```cpp
gcd(l, r)
gcd(l+1, r)
...
gcd(r, r)
```

右端点始终是 `r`。

---

### 2. `g[i]` 的转移才是本题灵魂

核心思想：

```cpp
所有以 i 结尾的后缀
=
单独的 a[i]
+
所有以 i-1 结尾的后缀接上 a[i]
```

对应公式：

$$
f_i(j)=\gcd(f_{i-1}(j),a_i)
$$

这就是本题的 DP 味道。

---

### 3. gcd 的变化次数很少，可以压缩

gcd 不会频繁乱跳。
它只会保持不变或变成当前值的因子。

因此可以记录变化点，而不是记录所有左端点。

---

### 4. 压缩状态不会丢失信息

如果一段起点在 `i-1` 时 gcd 都是 `x`，那么接上 `a[i]` 后，它们都会变成：

```cpp
gcd(x, a[i])
```

所以只存这一段的代表值和变化点就够了。

---

### 5. 类型范围必须认真看

`2^31` 不等于 `int` 能存。

`int` 最大是：

```cpp
2147483647
```

而：

```cpp
2^31 = 2147483648
```

所以这题数组值必须用 `long long`。
